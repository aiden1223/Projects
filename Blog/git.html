Git 與 Github隨手記

November 1, 2023 Aiden
Git指令：
  • 移動路徑：cd 路徑 (ex.cd /c) 或是cd +直接拖曳資料夾至Git Bash即可獲得路徑
  • 回上一層：cd ..
  • 顯示當前資料夾內列表：ls
  • 開新資料夾：mkdir 資料夾名稱
  • 開新檔案：touch 檔案名稱 (ex.touch style.css)
Git環境設定：
  • 查詢版本：git version
  • 查詢設定列表：git config --list
  • 輸入姓名：git config --global user.name "aiden"
  • 輸入email：git config --global user.email "aiden@gmail.com"
版控步驟-維護：
  1. cd至前次連結過的專案資料夾
  2. 增加全部檔案進入索引：git add .(.表示全部)
  3. 將索引提交到數據庫：git commit -m "修改註記"
  4. 將Git commit完的內容上傳到GitHub：git push
版控步驟-新建：
  1. cd至欲上傳GitHub的專案資料夾
  2. 在本地資料夾新增數據庫：git init (出現master代表有連結到Local Repository)
  3. 增加全部檔案進入索引：git add .(.表示全部)
  4. 將索引提交到數據庫：git commit -m "修改註記"
  5. 至Github先創建一個新的Repositories，取得"HTTPS網址"
  6. 連線到遠端數據庫：git remote add origin(遠端數據庫) "HTTPS網址"
  7. 將當前分支(master)的更改推送到遠端並建立上游關聯：git push --set-upstream origin(遠端數據庫) 分支名稱(例如main/master)
  8. 後續要將異動完的內容上傳到GitHub：git push
忽略檔案(指定某些檔案不進行版控)：
  • 產生文件：touch .gitignore
  • 開啟.gitignore.txt 記事本，在裡面輸入欲忽略的文件(ex.index.html、*.html、css/)
Git分支
  • 瀏覽目前處在分支：git branch
  • 查看其他commit紀錄的程式：git checkout 前四碼(git log取得)
  • 回到最新commit紀錄的程式：git checkout master
  • 創建新分支：git branch 分支名稱
  • 新分支若要push上Github：git push --set-upstream origin(遠端數據庫) 新分支名稱
  • 切換分支：git checkout 分支名稱
  • 合併分支：git merge 被合併分支名稱 (要先切回主要分支)
Git標籤
  • 建立tag前要先進行commit的動作
  • 建立標籤：git tag v1(標籤名稱)
  • 建立標籤(含備註)：git tag -am "備註" v1
  • 查詢標籤：git tag
  • 查詢詳細標籤：git tag -n
  • 切換標籤：git checkout v1
  • 刪除標籤：git tag -d v1
Git暫存
  • 建立暫存：git stash
  • 查詢暫存：git stash list
  • 還原暫存：git stash pop
  • 清除最新暫存：git stash drop
  • 清除全部暫存：git stash clear
  • 暫存可以跨分支使用
GitHub相關
  • 查詢遠端數據庫：git remote
  • 修改遠端數據庫名稱：git remote 原名稱 新名稱
  • 下載遠端數據庫：git pull
  • 建立遠端數據庫的分支(用意是在pull前先查看)：git fetch origin(遠端數據庫) 遠端分支名稱 (此時分支會多一個叫 FETCH_HEAD 的分支)
  • 查看欲pull的分支：git checkout FETCH_HEAD
申請GitHub個人token
  1. 點擊GitHub頭像選擇Settings
  2. 在左邊選單點擊Developer settings
  3. 點擊Personal access tokens->Tokens(classic)->Generate new token
  4. 點擊Generate new token(classic)
  5. 輸入名稱，設定期限，設定權限(若不確定可全勾)->產生token
指令大全：
  • 複製遠端數據庫：git clone 遠端數據庫網址
  • 增加檔案進入索引：git add 檔案名稱
  • 單一檔案取消索引：git reset HEAD 檔案名稱
  • 全部檔案取消索引：git reset HEAD
  • 還原檔案(到上次commit版本)：git checkout 檔案名稱
  • 還原目錄(到上次commit版本)：git reset --hard
  • 恢復單一檔案到最新 commit 狀態：git checkout 檔案名稱
  • 還原工作目錄與索引，會跟最後一次 commit 保持一樣：git reset --hard
  • 刪除最近一次 commit：git reset --hard HEAD^
  • 刪除錯了還原：git reset --hard ORIG_HEAD
  • 刪除最近一次 commit，但保留異動內容：git reset --soft HEAD^
  • commit 後想要補內容進去時：git commit --amend
  • 查詢狀態：git status
  • 顯示歷史紀錄：git log
  • 離開歷史紀錄：q
  • 重命名 Git 存儲庫中的分支：git branch -m 'old-branch-name' 'new-branch-name'
  • 如果git push沒有成功：Ctrl + C（Windows 或 Linux）或 Cmd + C（Mac）來中斷 git push 的執行。
  • 刪除Repository：進入該Repository->Settings->General最下面Danger Zone->Delete this repository
  • Git bash貼上方式：Ctrl+Shift+Ins
